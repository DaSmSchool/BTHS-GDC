PICO-8 is actually a little bit different from the other engines mentioned so far, as it's actually an all-in-one suite for making small sprite-based games! All of the tools for making 2D games are in PICO-8, including drawing graphics, making sound effects and music, and of course, writing code! The engine does have a pretty strict set of rules to abide by that imitates the limitations of older game consoles, but it's actually a pretty good way of making projects that avoid scope creep! It uses Lua as its programming language, which is similar to other languages like python for its English-like readability. If you really don't want to work with PICO-8's limitations, you *could* use [[Love2D]], but try this out first! It's worth it! Please! I'm begging you!

The PICO-8 also has a community of people making carts for it, and you can play pretty much all of them on the PICO-8 [BBS](https://www.lexaloffle.com/bbs/?cat=7#sub=2)!

If you haven't learned how to program yet, or if you have never made a game project before, check out the [[Lazy Devs]] Schmup tutorial series, it both introduces you to programming and (more importantly) orienting yourself at developing game projects.

If you *really* want to make a larger scale project, you should probably make a game with PICO-8 anyway. Unless you know what you are doing and have a plan, you need to make a small project to try to understand the process for organizing a large scale project.

The downloadable version that lets you run and make games natively for Windows/Mac/Linux/Raspberry Pi costs $15.00, though the HTML education edition is free, and has all of the features that PICO-8 carts can utilize. The only difference is that you can't export the music, graphics, executables outside of PICO-8, or access the in-engine online cart explorer.
