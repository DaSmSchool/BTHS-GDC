The larger starting guide has yet to be completed, so have a smaller set of steps that will get you started:

Ideally, you should start out with the [Schmup tutorial](https://www.youtube.com/watch?v=81WM_cjp9fo&list=PLea8cjCua_P3Sfq4XJqNVbd1vsWnh7LZd) by [[Lazy Devs]], but if you are at school, you can follow along some of the next steps here:

1. Head over to the [[PICO-8]] [education edition](https://www.pico-8-edu.com). It will look like an older computer, and that's kinda by design! PICO-8 carts and games are supposed to be smaller in scope and capabilities to help people wrap their head around the idea of programming and finishing their carts first, rather than trying to wander around all of the possibilities and approaches to making their projects with bigger engines like Unity, Unreal or Godot.
2. Once loaded in, type in ```install_demos```. All characters will show up as uppercase when typing in PICO-8, so don't use Shift or Caps Lock!
3. After it shows you a list of file names, type in ```cd demos``` to reach the folder with the installed demos.
4. If you ever want to bring up the list of installed demos, type ```ls```, and the console should show you that same list of file names.
5. If you found a cart you want to look around in, type ```load (name of the cart).p8``` (the .p8 part isn't necessary, though you can still type it if you have multiple files with the same name and different file extensions.)
6. The console should say that the cart has loaded. To run the cart at any time, hit Ctrl-R to run the program! The controls use the arrow keys, along with the Z and X keys that represent the O and X buttons in PICO-8.
7. To pause the program and/or switch between the console and editor views, hit the Escape key.
8. The first screen you will see in the editor view will likely have a bunch of code in it. Use the arrow keys or the mouse wheel to scroll through that code.
9. Play around with the code! You don't have to do anything fancy for this right now, just try and change some of the numbers that are in the code and see what happens. If there's a number that's next to some word like jump or jmp, that probably has something to do with jumping! Make your character/enemies faster, change whatever text you saw in the program/game, maybe change the positions of where some things show up, it's up to you! Try to get a sense of what the code in the editor looks like, and how it is structured, even if you don't fully understand it yet.
10. Check out some of the other tabs at the top right too, and see what those do! Change what the graphics might look like, maybe play around with some of the sounds, try making a beat, get familiar!

Again, the [Schmup Tutorial](https://www.youtube.com/watch?v=81WM_cjp9fo&list=PLea8cjCua_P3Sfq4XJqNVbd1vsWnh7LZd) will probably net you the best results when it comes to orienting you around the PICO-8 and game development for the first time, but the above should be fine too!
